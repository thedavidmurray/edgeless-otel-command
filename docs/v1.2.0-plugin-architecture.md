# v1.2.0 Plug-in Architecture — Design Doc

**Goal**: turn Edgeless OTel Command from a fixed dashboard into a platform where the community contributes panels, anomaly rules, themes, and alert sinks. Solo developer can keep maintaining the core; users can extend it without forking.

## Inspiration / prior art

| Tool | Lesson we take |
|------|----------------|
| **Obsidian** | Manifest + JS module, decentralized via GitHub. Trust through transparency. PR-reviewed registry. **This is closest to our model.** |
| **VS Code** | `contributes` block in manifest. Strict permissions. We borrow the "contributes" shape but keep it lighter. |
| **Winamp** | Visual skins as a shareable artifact. We do this for themes/layouts in v2.0. |
| **Grafana** | Panel plug-ins as the killer use case. We start here. |
| **Figma** | Iframe sandboxing. Defer to v2 — for now use the same trust model as Obsidian. |

## Architecture

### Plug-in = a folder with a manifest

```
my-edgeless-plugin/
├── manifest.json          # required
├── index.js               # required — entry point
├── README.md              # recommended
├── icon.png               # optional, 64x64
└── styles.css             # optional, scoped via shadow DOM in v2
```

Distribution: zipped, or `git clone`d to `userData/plugins/<plugin-id>/`.

### Manifest

```jsonc
{
  "id": "edgeless.plugin.service-map",         // dotted, unique
  "name": "Service Map",
  "version": "1.0.0",                          // semver
  "description": "Visual graph of services from parent-child span references.",
  "author": "edgeless-dev",
  "homepage": "https://github.com/edgeless-dev/edgeless-plugin-service-map",
  "license": "MIT",
  "minAppVersion": "1.2.0",
  "contributes": {
    "panels":       [ { "id": "service-map", "label": "Service Map" } ],
    "anomalyRules": [ { "id": "service-map.disconnect", "label": "Disconnected service" } ],
    "themes":       [ /* { "id": "...", "label": "...", "cssPath": "..." } */ ],
    "alertSinks":   [ /* future: { "id": "slack", "label": "Slack webhook" } */ ]
  }
}
```

`id` is the namespace. All entities the plug-in registers must use that prefix.

### Entry point

`index.js` exports a single function:

```js
// Pure-function registration. Called once at load time.
// Receives the host API; returns nothing. Throws on misuse.
export default function activate(edgeless) {
  edgeless.panels.register('service-map', {
    label: 'Service Map',
    render(container, ctx) {
      // container: HTMLElement we own (cleared between renders)
      // ctx: { traces, services, rollup, anomalies, settings, lib }
      container.innerHTML = '<div>Hello</div>';
    },
    destroy() {
      // optional cleanup
    },
  });

  edgeless.anomalies.register('service-map.disconnect', {
    label: 'Disconnected service',
    severity: 'warn',
    detect(ctx) {
      // ctx.traces, ctx.byService, ctx.timeWindowMin
      // return [{ kind, title, detail, service, traceID }] or []
      return [];
    },
  });
}
```

### The `edgeless` host API (v1 — stable)

```ts
interface EdgelessApi {
  app: {
    version: string;                  // "1.2.0"
    log(msg: string, level?: 'info'|'warn'|'error'): void;
  };

  panels: {
    register(id: string, def: PanelDef): void;
    list(): PanelInfo[];              // installed panels (read-only)
  };

  anomalies: {
    register(id: string, def: AnomalyDef): void;
    list(): AnomalyInfo[];
  };

  themes: {
    register(id: string, css: string): void;
  };

  router: {
    navigate(hash: string): void;     // "#/trace/<id>"
    current(): { view: string; params: Record<string,string> };
  };

  jaeger: {
    fetch(path: string): Promise<any>;  // proxied
    fetchTrace(id: string): Promise<Trace | null>;
  };

  lib: {
    fmtDur(us: number): string;
    fmtAge(us: number): string;
    hashColor(s: string): string;
    tagsToObj(tags: Tag[]): Record<string, any>;
  };

  storage: {
    // scoped to plug-in id
    get<T>(key: string): Promise<T | null>;
    set<T>(key: string, value: T): Promise<void>;
  };

  events: {
    on(event: 'tick'|'trace-selected'|'service-selected', cb: (data: any) => void): () => void;
  };
}

interface PanelDef {
  label: string;
  defaultPosition?: 'top-left'|'top-mid'|'top-right'|'bottom-left'|'bottom-mid'|'bottom-right';
  minSize?: { w: number; h: number };
  render(container: HTMLElement, ctx: PanelContext): void | Promise<void>;
  destroy?(): void;
}

interface PanelContext {
  traces: Trace[];
  byService: Record<string, Trace[]>;
  rollup: Record<string, ServiceRollup>;
  anomalies: Anomaly[];
  settings: Settings;
  serviceFilter: string | null;
  timeWindowMin: number;
  lib: EdgelessApi['lib'];
}

interface AnomalyDef {
  label: string;
  severity: 'crit'|'warn'|'ok';
  detect(ctx: AnomalyContext): Anomaly[];
}
```

The API surface stays SMALL. We don't expose DOM, file system, or arbitrary Electron access. Plug-ins can do anything within their `container`, but they don't reach outside it.

### Loading sequence

1. App startup: main.js scans `userData/plugins/` for folders.
2. For each, read `manifest.json`. Validate schema. Skip if invalid + log warning.
3. Pass manifest list to renderer via IPC.
4. Renderer constructs the `edgeless` API object.
5. For each enabled plug-in: dynamically `import()` its `index.js`, call `default(edgeless)`.
6. After all plug-ins registered, render dashboard with all panels (built-in + plug-in).

### Error containment

```js
async function loadPlugin(plugin) {
  try {
    const mod = await import(plugin.entryUrl);
    if (typeof mod.default !== 'function') throw new Error('no default export');
    mod.default(makeApi(plugin));
  } catch (e) {
    edgeless.app.log(`[${plugin.id}] failed to load: ${e.message}`, 'error');
    pluginRegistry.markFailed(plugin.id, e.message);
  }
}

function wrapPanelRender(plugin, renderFn) {
  return (container, ctx) => {
    try { return renderFn(container, ctx); }
    catch (e) {
      container.innerHTML = `<div class="plugin-error">⚠ ${plugin.id}: ${e.message}</div>`;
      edgeless.app.log(`[${plugin.id}] render error: ${e.stack}`, 'error');
    }
  };
}
```

### Built-in panels become plug-ins

To prove the design works, the existing panels (Throughput, Op Breakdown, Heatmap, Anomaly, Recent Traces) become "built-in plug-ins" loaded from `./built-ins/` at startup. Same API as community plug-ins. If they work, the design is sound.

## Community sharing (v1.3.0)

Registry repo `edgeless-otel-plugins/registry` with one file:

```jsonc
// plugins.json — fetched daily by the app
{
  "version": 1,
  "lastUpdated": "2026-05-20",
  "plugins": [
    {
      "id": "edgeless.plugin.service-map",
      "name": "Service Map",
      "description": "...",
      "repo": "edgeless-dev/edgeless-plugin-service-map",
      "tags": ["panel", "visualization"],
      "screenshot": "https://...",
      "minAppVersion": "1.2.0",
      "latestVersion": "1.0.0"
    }
  ]
}
```

Submission process (in registry's CONTRIBUTING.md):
1. Fork the registry repo
2. Add your plug-in entry to `plugins.json`
3. Open PR
4. Maintainer reviews: works as advertised, MIT/permissive license, no malicious code
5. Merge

The app fetches `registry/main/plugins.json` once a day. "Browse Plugins" UI shows cards. Click "Install" → app downloads the latest GitHub Release artifact from `<repo>/releases/latest`, validates manifest, extracts to `userData/plugins/<id>/`.
